# 📋 Change Log - Altivec Intelligence

This document summarizes the changes and improvements made to the **Altivec Intelligence** project in this session.

## 🚀 Key Improvements

### 1. `AICURLConnection` Library
- **Asynchronous Engine**: Implemented full non-blocking support using `NSThread`. The UI now remains responsive during transfers.
- **Tiger-Compatible Threading**: Uses `detachNewThreadSelector:` and `performSelectorOnMainThread:` to ensure 10.4 compatibility.
- **NSURLConnection Parity**: Added `+connectionWithRequest:delegate:`, `start`, and `cancel` methods.
- **Full Header Support**: Now parses and accumulates HTTP headers into a dictionary, allowing delegates to access full response metadata.
- **AIHTTPURLResponse Subclass**: Reliable status code and header storage for legacy systems where `NSHTTPURLResponse` is restricted.
- **Memory Safety**: Optimized for Tiger's autorelease pool behavior to prevent leaks on background threads.

### 2. `CURLmac` Application Refactoring
- **Polished UI**: Implemented a vertical stack layout for the `DownloadView` with Aqua-native feel.
  - **Single-line URL Entry**: Configured for horizontal scrolling and no wrapping (Tiger/Leopard compatible).
  - **Dynamic Footer**: A bottom area that switches between a `NSTextField` status label and an `NSProgressIndicator` bar during active downloads.
  - **Large Image Well**: A centerpiece `NSImageView` with Gray Bezel style to display downloaded content.
- **Shared Architecture**: Both `AICURLConnection` and `NSURLConnection` tabs now share the same `DownloadView` subclass for a consistent experience.
- **Componentization**: Extracted the Info/Libraries view into a reusable `KeyValueTableView` component.
- **Manual Menu Bar**: Implemented `MainMenu.h/.m` to build the entire system menu bar programmatically without NIBs, including the required Apple Menu hacks for legacy OS versions.
- **Error Handling**: Standardized error presentation using Cocoa sheets via `presentError:modalForWindow:`.

### 3. Deployment & Build System
- **Enhanced `altivec_deploy.sh`**:
  - Simplified arguments: `-d` for device, `-b` for build directory.
  - **Symbol Sync**: Now copies the entire build directory to the remote device via `scp -r` to assist debugger symbol discovery.
  - **Connection Pre-flight**: Verifies SSH connectivity and detects OS type (Mac vs. iPhone) before proceeding.
- **Multiples-of-4 Rule**: Audited and adjusted all UI frames, paddings, and column widths to strictly adhere to 4px or 8px grid alignments.
- **Cross-Platform Compatibility**: Expanded `CrossPlatform.h` with macros for:
  - Window Style Masks
  - Event Modifier Flags
  - Progress Indicator Styles
  - Application Delegate protocols

## 🛠 Build System Updates
- **Makefile Refinement**: Updated `libs/libcurl` and `apps/CURLmac` Makefiles to support new targets (`aicc`, `aicc-phone`) and include the new source files.
- **Quad-Fat Verification**: All changes verified to compile cleanly for `ppc`, `i386`, `x86_64`, and `arm64`.
