# 📋 Change Log - Altivec Intelligence

This document summarizes the changes and improvements made to the **Altivec Intelligence** project in this session.

## 🚀 Key Improvements

### 1. `AICURLConnection` Library
- **Asynchronous Engine**: Implemented full non-blocking support using `NSThread`. The UI now remains responsive during transfers.
- **Tiger-Compatible Threading**: Uses `detachNewThreadSelector:` and `performSelectorOnMainThread:` to ensure 10.4 compatibility.
- **NSURLConnection Parity**: Added `+connectionWithRequest:delegate:`, `start`, and `cancel` methods.
- **Full Header Support**: Now parses and accumulates HTTP headers into a dictionary, allowing delegates to access full response metadata.
- **AIHTTPURLResponse Subclass**: Reliable status code and header storage for legacy systems where `NSHTTPURLResponse` is restricted.
- **Memory Safety**: Optimized for Tiger's autorelease pool behavior to prevent leaks on background threads.
- **Human-Readable Sizes**: Added `XP_stringFromByteCount:` helper to provide "1.2 MB" style formatting on legacy systems.

### 2. `CURLmac` & `CURLphone` Evolution
- **Unified Architecture**: Both Mac and iPhone apps now share a consistent internal design.
  - **Tabbed Interface**: Three-tab layout (CURL, System, Libraries) for direct comparisons.
  - **DownloadManager**: Logic for coordinating UI updates and networking events.
  - **Programmatic UI**: Entirely NIB-free implementation for maximum portability and compatibility.
- **iPhone (CURLphone) Implementation**: 
  - Adapted the Mac "Quad-Fat" logic for iOS 4.3 through modern arm64 devices.
  - Implemented grouped table views for a native iOS "settings-style" feel.
  - Real-time progress updates via `UIProgressView` and centered status labels.
- **Code Consolidation (Kaizen)**: Drastically reduced file count for better maintainability.
  - **Mac**: Consolidated all logic into `main.m` and `DownloadWindowController.m`.
  - **iPhone**: Consolidated all logic into `main.m`, `DownloadViewController.m`, and `KeyValueTableViewController.m`.
  - Moved shared macros and protocols into `CrossPlatform.h` logic.

### 3. Deployment & Build System
- **Wait for Launch**: Implemented an automated "Launch and Attach" debugging workflow in `altivec_deploy.sh`.
- **Automatic Resource Injection**: The build system now automatically copies `cacert.pem` into both Mac and iPhone app bundles to ensure SSL verification works out-of-the-box.
- **Remote Pre-flight Improvements**: SSH connection checks are now more robust, correctly handling vintage Mac handshakes.
- **Cross-Platform Macros**: Consolidated version-gated macros (10.6, 10.12, 10.14) into a single, verified header.

## 🛠 Build System Updates
- **Phone Build Engine**: Created `altivec_common_phone.mk` to handle universal iOS builds (armv7 + arm64).
- **Library Parity**: Ensured `libcurl` and its dependencies build cleanly for all Apple architectures.
